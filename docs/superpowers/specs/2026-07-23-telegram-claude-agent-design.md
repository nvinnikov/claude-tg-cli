# tg-claude — Telegram-бот как фронтенд к Claude Code на локальной машине

Дата: 2026-07-23
Статус: дизайн утверждён, план не написан

## Задача

Дать возможность управлять рабочей машиной (mac) из Telegram с телефона: завести
сессию, поставить задачу, посмотреть состояние сервисов/кластера/репозиториев.
Всё, что делается руками в терминале, должно быть доступно из чата.

## Решения и их обоснование

| Решение | Выбор | Почему |
|---|---|---|
| Мозг агента | Claude Agent SDK (Python, `claude-agent-sdk`) | Это Claude Code как библиотека: те же встроенные инструменты, MCP, скиллы, resume сессий. Свой agent loop писать не нужно. |
| Хост | Локальный mac | Целевая машина и есть mac: на ней уже настроены рабочие доступы, клоны репозиториев и локальные стенды. |
| Транспорт | Telegram Bot API, long polling | Мак за NAT, вебхук потребовал бы туннеля. |
| Сессии | По одной на topic супергруппы | Позволяет вести несколько задач параллельно, у каждой свой `cwd` и контекст. |
| Права | Whitelist команд + подтверждение в чате для остального | Полный доступ без вопросов слишком рискован; чистый whitelist упирается в лимиты каждые пару минут. |
| Вывод | Компактный прогресс + итог | Одно редактируемое сообщение со строкой на tool-call, финальный ответ отдельно. |

Рассмотрен и отклонён [Centaur](https://github.com/paradigmxyz/centaur): Slack вместо
Telegram, исполнение в одноразовом k8s-сандбоксе вместо целевой машины, вес
(k3s + Postgres + 1Password + прокси) несопоставим с однопользовательской задачей.
Взятая оттуда идея на будущее — credential boundary (см. «Вне объёма»).

## Архитектура

```
Telegram (супергруппа с topics)
   │  long polling
   ▼
bot.py ─ aiogram 3, asyncio
   ├─ store.py       thread_id → {session_id, cwd, статус}
   ├─ runner.py      ClaudeSDKClient, стрим сообщений
   ├─ permissions.py can_use_tool: whitelist / denylist / запрос в чат
   └─ render.py      события → текст, markdown→HTML, чанкинг
        ▼
   shell на маке (под твоим юзером)
```

Демон под `launchd` (`KeepAlive=true`, `RunAtLoad=true`), логи в файл с ротацией.

### Компоненты

**store.py** — SQLite. Таблица `sessions(thread_id PK, session_id, cwd,
progress_msg_id, updated_at)`. Одна запись на topic. При старте бота читается,
сессии восстанавливаются через `resume`.

**runner.py** — обёртка над `ClaudeSDKClient`. На каждый topic — свой клиент с
`ClaudeAgentOptions(cwd=..., resume=session_id, can_use_tool=..., allowed_tools=[...])`.
Читает `receive_response()`, отдаёт наверх поток событий. Прерывание — `client.interrupt()`
с обязательным дренажом буфера до `ResultMessage` перед следующим запросом.

**permissions.py** — колбэк `can_use_tool(tool_name, input_data, context)`.
Порядок проверок:

1. Deny-лист (regex по команде Bash + по путям для Write/Edit). Матч → `PermissionResultDeny`,
   в чат уходит уведомление. Кнопкой не обходится.
2. Whitelist из `allowlist.yaml`. Матч → `PermissionResultAllow`.
3. Иначе — сообщение в topic с текстом команды и inline-кнопками
   `Разрешить раз` / `В whitelist` / `Запретить`. Колбэк ждёт ответа с таймаутом
   (по умолчанию 5 минут); по таймауту — deny.

`Read`/`Grep`/`Glob` идут в `allowed_tools` и до колбэка не доходят вовсе.
`permission_mode` остаётся `default`, иначе `Bash` тоже минует колбэк.

**render.py** — сборка прогресс-сообщения, конвертация markdown в Telegram HTML,
нарезка длинных ответов по 4000 символов, оборачивание кода в `<pre>`.

**bot.py** — aiogram-роутер: фильтр по `allowed_user_id`, команды, обработчики
callback-кнопок, диспетчеризация текста в нужную сессию по `message_thread_id`.

## Поведение

### Команды

| Команда | Действие |
|---|---|
| `/new` | Сбросить сессию в текущем topic (новый `session_id`) |
| `/cd <path>` | Сменить `cwd` сессии; путь валидируется на существование |
| `/pwd` | Показать текущий `cwd` и `session_id` |
| `/stop` | Прервать текущий прогон в этом topic |
| `/status` | Список всех topic'ов: `cwd`, занят/свободен, что выполняется |

Любое другое сообщение — промпт в сессию текущего topic.

### Прогресс

На каждый прогон создаётся одно сообщение, редактируемое не чаще раза в 3 секунды
(лимит Telegram на `editMessageText`):

```
⏳ Работаю…
▸ Bash: git status
▸ Read: services/oms/AGENTS.md
▸ Bash: make test
```

По завершении оно схлопывается в `✅ 3 шага, 47s` (или `⛔ прервано` / `❌ ошибка`),
а финальный ответ приходит отдельным сообщением.

### Безопасность

- Единственный разрешённый `user_id` в конфиге; всё остальное молча игнорируется.
- Deny-лист поверх whitelist: `rm -rf /`, `dd`, форк-бомбы, `--context prod` для
  `kubectl`, `git push --force`.
- Токен бота и конфиг — вне git, права `600`.
- Известное ограничение: агент работает под твоим юзером и имеет доступ ко всем
  токенам в `~`. Изоляции нет — это осознанный размен на доступ к рабочей машине.

## Вне объёма

Веб-UI, мультиюзерность, очередь задач, докеризация, credential boundary
(плейсхолдеры + подстановка секретов на прокси — понадобится, только если бот
получит прод-доступ).
